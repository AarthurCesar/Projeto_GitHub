Comando git merge

O comando git merge é usado para unir o conteúdo de uma branch com a branch atual.

🔀 Exemplo prático

Você está na branch main e quer unir o conteúdo da branch login nela:

git checkout main
git merge login

Esse comando pega tudo o que foi feito na branch login e aplica na main.

⚔️ Conflitos de merge

Se houver arquivos modificados nas duas branches, o Git pode não saber qual versão manter. Isso é um conflito.

Quando isso acontece:

  • O Git marca o conflito no arquivo
  • Você deve abrir o arquivo e escolher o que manter
  • Depois, finalize com:
git add .
git commit -m "Resolvido conflito do merge"

🔁 Merge vs Rebase

O merge cria um novo commit com o histórico das duas branches. Já o git rebase reorganiza os commits. Para projetos simples, merge é mais seguro e fácil de entender.

✔️ Dica de fluxo comum

  1. Você está na branch principal (ex: main)
  2. Alguém terminou a feature na branch funcionalidade
  3. Para unir:
git checkout main
git pull
git merge funcionalidade
