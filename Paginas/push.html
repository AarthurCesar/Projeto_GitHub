Comando git push

O comando git push é usado para enviar suas alterações locais (commits) para um repositório remoto, como o GitHub.

🚀 Enviar commits para o GitHub

Comando mais comum para enviar tudo que você fez localmente:

git push origin main

  • origin → o nome do repositório remoto (geralmente padrão)
  • main → a branch principal (pode ser master ou outras feature, bugfix, hotfix, release e beta etc...)

🔐 Primeira vez usando push?

Você precisa configurar o repositório remoto antes:

git remote add origin https://github.com/usuario/repositorio.git

Se for a primeira vez que está empurrando essa branch, pode ser necessário:

git push -u origin main

O -u (ou --set-upstream) faz com que o Git lembre para onde empurrar da próxima vez.

🧠 Dica

Depois disso, nos próximos pushes, basta:

git push

❗ Problemas comuns

  • Erro de autenticação: confirme se está logado com SSH ou token.
  • Branch divergente: pode exigir um git pull antes.
  • Branch inexistente: verifique se você digitou corretamente o nome da branch.
