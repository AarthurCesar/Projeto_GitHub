Comando git pull

O comando git pull é usado para atualizar seu repositório local com as alterações que estão no repositório remoto (ex: GitHub).

📥 Baixar e mesclar alterações

Esse comando faz duas coisas:
  1. Baixa as alterações do GitHub
  2. Tenta mesclar essas mudanças na sua branch atual

git pull origin main

  • origin → repositório remoto
  • main → nome da branch (pode ser master ou outra)

📌 Situações comuns de uso

  • Antes de começar a programar
  • Quando alguém fez alterações no GitHub
  • Para garantir que está com a versão mais atualizada

⚠️ Possíveis conflitos

Se você alterou arquivos que também foram alterados no GitHub, pode surgir um conflito. O Git vai pedir que você resolva manualmente antes de continuar.

🛠 Exemplo prático

Imagine que alguém fez um commit no GitHub. Para trazer para sua máquina:

git pull origin main

✔️ Depois de resolver conflitos

Após corrigir os conflitos:

git add .

git commit -m "Resolvi conflitos do pull"
